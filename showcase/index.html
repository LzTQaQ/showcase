Chapter 01

超声设备 AI 引擎

AI 驱动的企业级超声设备知识问答系统 — 从 RAG 知识库到多端部署

Background

为什么做这个项目？

医疗设备行业知识密集，产品线繁多，销售和售后团队需要快速掌握大量技术参数和竞品信息。传统培训方式效率低，知识查找困难。

行业痛点

迈瑞内部销售与售后团队需要面对GE、飞利浦、西门子等多家品牌的数百款超声设备型号。传统培训依赖纸质手册和经验传授，知识获取效率低、成本高。

技术选型

选择 Dify + DeepSeek 作为AI引擎，FastAPI 作为API网关，React + 微信小程序覆盖 Web 和移动端。核心设计理念：安全合规、多端一致、易维护。

项目成果

一个生产级、已部署的企业内部AI知识问答系统。覆盖 Web 网页端、微信小程序端，支持 AI 对话、设备参数查询、文件上传分析、对话管理等功能。包含完整的认证授权、审计日志、合规过滤体系，已通过 Docker 部署至个人 UGREEN NAS（绿联 DXP4800Plus）。网页端（PC + 移动）已上线并开始小范围灰度测试，微信小程序端开发完成但因 ICP 备案暂缓发布

Tech Highlights

技术亮点

从本地数据向量化到公网穿透，从 RAG 知识库到前端体验优化，全链路技术细节

RAG 知识库构建

基于 Dify 平台搭建企业级 RAG 知识库，涵盖迈瑞、GE、飞利浦等 12 个品牌的超声设备技术文档。知识库总规模 14GB，21,410 个知识块，覆盖产品规格书、探头参数、临床应用指南等多维度内容。

  • Dify 编排平台管理文档嵌入与检索流程
  • 多品牌竞品参数知识库（6+ 主流品牌全覆盖）
  • BM25 关键词 + TF-IDF 语义向量双路混合检索
  • 6 类业务场景自动识别：参数对比/概念解读/设备查询等

Ollama 本地化 Embedding

使用 Ollama 在 NAS 本地运行 Embedding 模型，对医疗设备文档进行高精度向量化。所有数据在本机完成向量化处理，无需上传至外部 API，兼顾精度与数据安全。

  • 本地 Embedding 模型，数据不外传
  • 医疗专业文档的领域适配向量化
  • Docker 部署在 NAS，与 Dify RAG 引擎无缝集成
  • 支持增量文档更新与重向量化

腾讯云反向代理 + Tailscale 公网穿透

前端部署在家庭 NAS，通过腾讯云轻量服务器 + Tailscale WireGuard 隧道实现公网访问。非标准端口 8765 规避 ICP 备案限制，SSE 流式响应完美支持。

  • 腾讯云轻量服务器 2C4G（公网 IP）
  • Tailscale WireGuard 隧道加密传输
  • Nginx 反向代理配置 SSE 支持（proxy_buffering off）
  • 非标准端口 :8765，无需 ICP 备案即可公网服务

前端性能与体验优化

React 19 + TypeScript + Vite 8 + Tailwind v4 技术栈，针对移动端和桌面端分别优化。Apple 设计风格，暗色模式支持，流式 AI 输出体验。

  • 移动端 100dvh 适配，视觉全屏无留白
  • 响应式字体（移动 24px / 桌面 34px）
  • 对话气泡宽度自适应（移动端 90%）
  • Zustand 状态管理 + React Router 路由
  • Apple 风格 UI 设计系统 + 暗色模式
Compliance & Security

合规与安全体系

全链路安全防护，确保AI输出合规、数据安全、可审计

AI 输出合规过滤

后端对每一条AI输出进行15类禁用话术扫描，命中则拦截并替换为合规提示。

  • “唯一可行”、“必须选用”、“排他性”
  • “指定机型”、“推荐锁定”、“控标”
  • “独家满足”、“只能用迈瑞”

审计日志系统

每次AI调用都记录完整审计链，包含用户ID、时间戳、输入输出摘要。日志轮转保留6个月。

  • 用户 ID + 时间戳 + IP 地址
  • 输入摘要（前200字）+ 输出摘要
  • 文件操作完整审计链
  • 日志不可篡改

身份认证与授权

JWT Token认证 + 白名单预审机制，管理员审批新用户。Token支持服务端撤销。

  • 工号 + 姓名实名登录
  • 管理员预审 + 审批
  • JWT 服务端撤销
  • 无游客 / 外部访问入口

安全防护体系

多层防护：速率限制、CORS策略、安全头、文件魔数校验、请求体大小限制。

  • SlowAPI 速率限制（10-30/min）
  • CSP / HSTS / X-Frame-Options
  • 文件魔数校验防伪造
  • 10MB 请求体限制

千里马招标智能监测系统

基于浏览器自动化的每日超声设备招标信息监控
背景：迈瑞销售期间负责达州、巴中地区，每天需手动查看千里马招标网上彩超相关招标信息，按地区和日期筛选。耗时且容易遗漏。
技术实现：Chrome CDP 远程控制 → 自动登录千里马高级会员 → 4组查询（当天/昨天 × 达州/巴中）→ AI 分析命中结果 → 飞书推送。每次操作间隔 1-3秒模拟真人，避免触发风控。
监测成果：
• 达州市第一人民医院 198万元彩超×2批次
• 渠县人民医院 472万元超声设备
• 通川区中医院 床旁彩超机 20万元

微信 Windows 客户端 GUI 自动化

PostMessage 突破 Qt5+DirectUI 重金属，实现微信自动发送
背景：微信 Windows 客户端使用 Qt5 框架 + 自研 DirectUI 渲染，标准 uiautomation 找不到任何子控件，所有常规方案全部失效。
技术路线迭代：
✖ uiautomation → 找不到子控件
✖ keybd_event → 被现代 Windows/微信过滤
✖ SendInput → 需要前台窗口权限
✔ PostMessage WM_CHAR → Qt5 正确路由到焦点控件

招标参数品牌归属判定

RAG 知识库 + BM25+TF-IDF 双路混合检索
背景：医疗招投标中，拿到一份数百条技术参数的招标文件，第一件事是判断这是哪个品牌的“锁标参数”。人工翻资料比对通常需要 1-2 天。
方案：搭建了 12 品牌竞品知识库（14GB，21410 个知识块），开发 BM25+TF-IDF 双路混合 RAG 检索引擎。未见过的招标参数，5 分钟完成品牌归属判定，附带逐条证据链分析报告。
